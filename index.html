⚡ Connect to Supabase

Follow these steps once to link your live database. All survey responses will be stored and your dashboard will update in real time.

1

Create a free Supabase project

Go to supabase.com → New Project. Choose a name (e.g. ncwsc-survey) and a strong database password.

2

Run this SQL to create the responses table

In Supabase → SQL Editor, paste and run:

CREATE TABLE survey_responses (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at timestamptz DEFAULT now(),
  customer_type text, region text,
  reliability int, interruptions text, resolution_speed text,
  water_quality int, safety_confidence int,
  billing_accuracy int, payment_options int, charges_fair int,
  communication int, responsiveness int, online_access int,
  professionalism int, issue_resolution int,
  overall int, nps text, suggestions text
);
ALTER TABLE survey_responses ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow anonymous inserts" ON survey_responses FOR INSERT WITH CHECK (true);
CREATE POLICY "Allow anonymous reads" ON survey_responses FOR SELECT USING (true);
3

Get your Project URL & anon key

Go to Settings → API in your Supabase project. Copy the Project URL and anon public key below.

4

Deploy to Vercel

Go to vercel.com → Add New Project → upload this HTML file (or push to GitHub first). Add SUPABASE_URL and SUPABASE_ANON_KEY as Environment Variables, or just paste them below for a quick start.
